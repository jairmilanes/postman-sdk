EnvironmentManager

Creates a new environment instance and manage it's variables.

You can add & remove keys programmatically so it is easy to integrate into your build scripts and update them during deploys.

Variables can be used inside collection items by referencing each variable as {{variable}} in your collection item properties.

This environments can then be switched in the Postman SDK allowing to easily test different environments.

Use this to create multiple environments instead of creating one collection item per environment.

.add(key, valueopt, typeopt, enabledopt) → {number}

Adds a new variable to this environment. You can either provide a object containing all the properties or pass each one to the method call.

Parameters:
key ( string | object )

The variable name or the variable object. If object is provided, ignore value

value ( string ) <optional>

The variable value, ig object was provided as the first parameter, this can be ignored

type ( string ) <optional>

The variable type (string, number, boolean)

enabled ( boolean ) <optional>

A flag tallying if the variable is enabled or disabled by default

Source:
  • environment-manager.js, line 70
Example
const manager = new EnvironmentManager('MY_ENVIRONMENT')

// Use single properties
manager.add('HOST', 'myhost.com', 'string', true)

// Use property object
manager.add({
   key: 'HOST',
   value: 'myhost.com',
   type: 'string',
   enabled: true
})
